# คู่มือการใช้งาน Posfont (POS)

คู่มือนี้อธิบายวิธีใช้งานระบบ POS สำหรับพนักงานร้านอาหาร ครอบคลุมการสั่งอาหาร การจัดการเมนู การรายงาน และการสำรอง/กู้คืนข้อมูล

> **อ่านบน GitHub:** ใช้สารบัญด้านล่างเพื่อข้ามไปยังหัวข้อที่ต้องการ  
> **ในแอป POS:** เปิดเมนู **คู่มือ** หรือไปที่ `/manual` แล้วเลือก **ไทย** (EN | ไทย ที่แถบเมนู)

## 1. เริ่มต้นใช้งาน

### เข้าสู่ระบบ

1. เปิดเบราว์เซอร์ไปที่หน้า POS ของร้าน
2. คลิก **Sign in** ที่แถบเมนูด้านบน
3. กรอก **Username** และ **Password** ที่ได้รับจากผู้ดูแลระบบ
4. คลิก **Sign in**

> หากยังไม่ได้เข้าสู่ระบบ คุณยังใช้งาน POS ได้บางส่วน (เช่น โต๊ะ ออเดอร์) แต่เมนู **Users** จะใช้ได้หลังเข้าสู่ระบบเท่านั้น

### ออกจากระบบ

คลิก **Sign out** ที่แถบเมนูด้านบน

### ภาษา


ใช้ปุ่ม **EN | ไทย** ที่แถบเมนูด้านบนเพื่อเปลี่ยนภาษาหน้าจอ ระบบจำการเลือกไว้

## 2. เมนูหลัก

แถบเมนูด้านบนมีลิงก์สำคัญดังนี้:

| เมนู | ความหมาย |
|------|----------|
| **Orders** | รายการออเดอร์ทั้งหมด |
| **New order** | สร้างออเดอร์ใหม่ |
| **Foods** / **Add food** | รายการเมนู / เพิ่มเมนู |
| **Categories** / **New category** | หมวดหมู่อาหาร |
| **Kitchens** / **Kitchen prep** / **Add kitchen** | ครัว / หน้าจอเตรียมอาหาร |
| **Zones** / **Add zone** | โซนที่นั่ง |
| **Tables** / **Add table** | โต๊ะ |
| **Reports** | รายงานและสำรองข้อมูล |
| **User manual** | คู่มือการใช้งาน (ไทย/English) |
| **Users** | จัดการผู้ใช้ (ต้องเข้าสู่ระบบ) |

## 3. โต๊ะและโซน

### โซน (Zones)

โซนคือพื้นที่ในร้าน เช่น ชั้น 1, ลานกลางแจ้ง, VIP

- ไปที่ **Zones** เพื่อดูรายการโซน
- คลิก **Add zone** เพื่อเพิ่มโซนใหม่
- คลิก **Edit** เพื่อแก้ไข หรือ **Delete** เพื่อลบ (ถ้ามี)

### โต๊ะ (Tables)

แต่ละโต๊ะต้องอยู่ในโซนใดโซนหนึ่ง

1. ไปที่ **Tables**
2. ใช้ช่อง **Search** ค้นหาด้วยรหัสโต๊ะ
3. คลิก **Add table** เพื่อเพิ่มโต๊ะ
4. จากรายการโต๊ะ สามารถ:
   - สร้างออเดอร์ใหม่สำหรับโต๊ะนั้น
   - แก้ไขข้อมูลโต๊ะ
   - ชำระเงิน (Pay) สำหรับออเดอร์ที่ค้างอยู่
   - ลบโต๊ะ (ถ้าไม่มีข้อจำกัดจากระบบ)

---

## 4. ออเดอร์

### สร้างออเดอร์ใหม่

1. คลิก **New order** หรือสร้างจากหน้า **Tables**
2. เลือกโต๊ะ
3. เพิ่มรายการอาหาร (lines) พร้อมจำนวนและหมายเหตุ
4. บันทึกออเดอร์

### ดูและจัดการออเดอร์

ไปที่ **Orders**:

- **Search** — ค้นหาด้วยเลขออเดอร์
- **Show only not done** — แสดงเฉพาะออเดอร์ที่ยังไม่เสร็จ
- คลิกเลขออเดอร์เพื่อ **Edit** แก้ไขรายละเอียด
- เปลี่ยนสถานะแต่ละรายการ (เช่น รอทำ / ทำเสร็จ / ยกเลิก)
- **Pay** — ชำระเงิน (เงินสด, QR PromptPay, บัตรเครดิต ฯลฯ ตามที่ระบบรองรับ)

### สถานะออเดอร์ที่สำคัญ

| สถานะ | ความหมาย |
|-------|----------|
| **Done** | ออเดอร์เสร็จสมบูรณ์ |
| **Cancel** | ออเดอร์ถูกยกเลิก |
| **Paid** | ชำระเงินแล้ว |

## 5. เมนูอาหารและหมวดหมู่

### หมวดหมู่ (Categories)

1. ไปที่ **Categories**
2. เพิ่ม แก้ไข หรือลบหมวดหมู่ เช่น อาหารจานหลัก, เครื่องดื่ม, ของหวาน

### เมนู (Foods)

1. ไปที่ **Foods**
2. คลิก **Add food** เพื่อเพิ่มเมนู
3. กำหนดชื่อ ราคา หมวดหมู่ และครัวที่รับผิดชอบ
4. แก้ไขหรือลบเมนูจากรายการได้

> ควรตั้งค่าหมวดหมู่และครัวให้ครบก่อนเปิดร้าน เพื่อให้การสั่งอาหารและหน้าจอครัวทำงานถูกต้อง

---

## 6. ครัวและหน้าจอเตรียมอาหาร

### ครัว (Kitchens)

- ไปที่ **Kitchens** เพื่อดูและจัดการครัว (เช่น ครัวร้อน, ครัวเครื่องดื่ม, บาร์)
- แต่ละเมนูอาหารควรผูกกับครัวที่ถูกต้อง

### Kitchen prep (หน้าจอเตรียมอาหาร)

ใช้สำหรับพ่อครัว/แม่ครัว:

1. ไปที่ **Kitchen prep**
2. เลือกครัวจากรายการ (หรือเลือก **All kitchens** เพื่อดูทั้งหมด)
3. ระบบแสดงเฉพาะรายการที่มีสถานะ **WAIT** (กำลังทำ) ของออเดอร์ที่ยังไม่ชำระและมีโต๊ะ
4. เมื่อทำอาหารเสร็จ คลิก **Finish cooking**
5. หน้าจอจะรีเฟรชอัตโนมัติทุกไม่กี่วินาที หรือคลิก **Refresh** ได้ทันที

## 7. รายงาน

ไปที่ **Reports** แล้วเลือก **Daily report**

### รายงานรายวัน

1. เลือก **Start date** และ **End date**
2. คลิก **Refresh**
3. ดูสรุปยอดขาย จำนวนออเดอร์ วิธีชำระเงิน และรายละเอียดแต่ละออเดอร์

**หมายเหตุสำคัญ:**

- ตัวเลขอิงจากออเดอร์ที่ **ชำระเงินแล้ว** ในช่วงวันที่เลือก
- **Cash received** และ **Change returned** นับเฉพาะการชำระด้วย **เงินสด**
- หากเซิร์ฟเวอร์รายงานไม่พร้อม ระบบอาจประมาณยอดจากรายการออเดอร์ในเครื่อง

## 8. สำรองและกู้คืนข้อมูล (Backup)

ไปที่ **Reports → Backup**

หน้านี้ใช้สำรองและกู้คืนข้อมูลทั้งหมดของ POS บนเซิร์ฟเวอร์

### ข้อมูลที่รวมในไฟล์สำรอง

| ประเภท | รวมในไฟล์สำรองเสมอ | กรองตามวันที่ได้ |
|--------|---------------------|------------------|
| โซน (Zones) | ✓ | — |
| โต๊ะ (Tables) | ✓ | — |
| ครัว (Kitchens) | ✓ | — |
| หมวดหมู่อาหาร | ✓ | — |
| เมนูอาหาร (Foods) | ✓ | — |
| ออเดอร์ (Orders) | ✓ (หรือกรอง) | ✓ |

---

### 8.1 ส่งออกข้อมูล (Export)

**ขั้นตอน:**

1. ไปที่ **Reports → Backup**
2. ในส่วน **Export** กำหนดช่วงวันที่ออเดอร์ (ถ้าต้องการ):
   - **From** — วันเริ่มต้น (วันที่สั่งอาหาร)
   - **To** — วันสิ้นสุด
   - คลิก **All orders** เพื่อล้างช่วงวันที่และส่งออกออเดอร์ทั้งหมด
3. คลิก **Create backup**
4. รอจนระบบสร้างไฟล์เสร็จ จะแสดง:
   - ชื่อไฟล์
   - ขนาดไฟล์
   - จำนวนออเดอร์ในไฟล์
   - ช่วงวันที่ (ถ้ามี)
5. คลิก **Download ZIP** เพื่อดาวน์โหลดไฟล์ `.zip` เก็บไว้ในที่ปลอดภัย

**กฎการเลือกช่วงวันที่ออเดอร์:**

| การตั้งค่า | ผลลัพธ์ |
|------------|---------|
| ว่างทั้ง From และ To | ส่งออกออเดอร์ **ทั้งหมด** |
| กรอกเฉพาะ From | ออเดอร์ตั้งแต่วันนั้นเป็นต้นไป |
| กรอกเฉพาะ To | ออเดอร์จนถึงวันนั้น |
| กรอกทั้ง From และ To | ออเดอร์ในช่วงวันที่ (รวมวันเริ่มและวันสิ้นสุด) |

> วันที่อิงตาม **order date** (วันที่สั่งอาหาร) ในรูปแบบปฏิทินท้องถิ่น (yyyy-MM-dd)  
> ข้อมูล master (โซน โต๊ะ ครัว หมวดหมู่ เมนู) จะถูกส่งออก **ครบทุกครั้ง** ไม่ว่าจะกรองออเดอร์อย่างไร

**รูปแบบไฟล์:** ZIP ที่มี JSON อยู่ภายใน สร้างและเก็บบนเซิร์ฟเวอร์ POS

---

### 8.2 นำเข้าข้อมูล (Import)

> ⚠️ **คำเตือน:** การนำเข้าจะ **ลบข้อมูลเดิมทั้งหมด** บนเซิร์ฟเวอร์ POS แล้วแทนที่ด้วยข้อมูลจากไฟล์สำรอง

**ข้อมูลที่ถูกลบและแทนที่:**

- โซน (Zones)
- เครื่องพิมพ์ (Printers)
- หมวดหมู่อาหาร
- ครัว (Kitchens)
- โต๊ะ (Tables)
- เมนู (Foods)
- ออเดอร์ (Orders)

**ขั้นตอน:**

1. ไปที่ **Reports → Backup**
2. ในส่วน **Import** คลิกเลือกไฟล์:
   - ไฟล์ `.zip` จากการ Export
   - หรือไฟล์ `.json` โดยตรง
3. ติ๊กถูกที่ช่อง **I understand all current data will be replaced.**  
   (ฉันเข้าใจว่าข้อมูลปัจจุบันทั้งหมดจะถูกแทนที่)
4. คลิก **Import & replace database**
5. รอจนเสร็จ ระบบจะแสดงจำนวนรายการที่กู้คืน เช่น โซน โต๊ะ เมนู ออเดอร์

**ข้อควรระวังก่อน Import:**

- สำรองข้อมูลปัจจุบันก่อนเสมอ (Export แล้ว Download ZIP)
- แจ้งพนักงานให้หยุดใช้งาน POS ชั่วคราวระหว่างนำเข้า
- ใช้ไฟล์สำรองจากเซิร์ฟเวอร์เดียวกันหรือที่ทราบว่าเข้ากันได้
- อย่านำเข้าไฟล์ที่ไม่แน่ใจว่ามาจากไหน

---

### 8.3 แนวทางใช้งาน Backup ที่แนะนำ

| สถานการณ์ | วิธีที่แนะนำ |
|-----------|--------------|
| สำรองประจำวัน | Export ทุกวันหลังปิดร้าน → Download ZIP → เก็บใน USB/Cloud |
| ย้ายข้อมูลไปเครื่องใหม่ | Export จากเครื่องเก่า → Import บนเครื่องใหม่ |
| สำรองเฉพาะออเดอร์ช่วงเวลา | กำหนด From/To แล้ว Export (master data ยังรวมครบ) |
| กู้คืนหลังข้อมูลเสียหาย | ใช้ไฟล์ ZIP ล่าสุดที่สำรองไว้ → Import |

## 9. จัดการผู้ใช้

> ต้อง **เข้าสู่ระบบ** ก่อน จึงจะเข้าเมนู **Users** ได้

### เพิ่มผู้ใช้

1. ไปที่ **Users**
2. กรอก Username, Password, Display name
3. กำหนด Roles (เช่น `ADMIN`, `STAFF`) คั่นด้วยจุลภาค
4. คลิก **Create user**

### จัดการผู้ใช้ที่มีอยู่

- ดูรายชื่อผู้ใช้ บทบาท (Roles) และสถานะ Enabled
- เปิด/ปิดการใช้งาน (Enable/Disable)
- คลิก **Refresh** เพื่อโหลดข้อมูลใหม่

## 10. ลูกค้าสั่งอาหารเอง (Guest)

ลูกค้าสามารถสั่งอาหารผ่านลิงก์พิเศษ (QR code ที่โต๊ะ) โดยไม่เห็นเมนูพนักงาน

- หน้า Guest จะไม่แสดงแถบเมนูด้านบนของพนักงาน
- ลูกค้าเลือกเมนูและส่งออเดอร์
- หลังสั่งสำเร็จจะเห็นหน้ายืนยันออเดอร์
- พนักงานตรวจสอบและดำเนินการต่อจากหน้า **Orders** หรือ **Kitchen prep** ตามปกติ

## 11. คำถามที่พบบ่อย

### Q: Export แล้วไม่เห็นปุ่ม Download ZIP
**A:** ต้องคลิก **Create backup** ให้สำเร็จก่อน ระบบจึงจะแสดงรายละเอียดไฟล์และปุ่มดาวน์โหลด

### Q: เลือก From มากกว่า To แล้ว Export ไม่ได้
**A:** วันที่ **From** ต้องไม่เกิน **To** ระบบจะแจ้งข้อผิดพลาด

### Q: Import แล้วออเดอร์วันนี้หาย
**A:** เป็นไปตามการทำงาน — Import แทนที่ฐานข้อมูลทั้งหมด หากไฟล์สำรองไม่มีออเดอร์วันนี้ ข้อมูลจะหายไป ควร Export สำรองก่อน Import เสมอ

### Q: รายงานรายวันไม่ตรงกับเงินในลิ้นชัก
**A:** ตรวจสอบว่าออเดอร์ถูกทำเครื่องหมาย **Paid** แล้ว และแยกยอดเงินสดจาก QR/บัตรเครดิต — รายงานแยกประเภทการชำระเงินไว้

### Q: หน้า Kitchen prep ว่าง
**A:** ตรวจสอบว่ามีออเดอร์ที่ยังไม่ชำระ มีโต๊ะ และรายการอาหารอยู่ในสถานะ **WAIT** หรือไม่

### Q: ลืมรหัสผ่าน
**A:** ติดต่อผู้ดูแลระบบที่จัดการบัญชีผู้ใช้บน API เซิร์ฟเวอร์ POS
